## Поэтапное описание

### Интерфейс создания занятия
Диалог начала занятий `lesson_dialog` переведен на Material. Исправлена цветовая схема: теперь фон всегда белый, а текст контрастный, что решило проблему нечитаемых айтемов в меню

```kotlin
bottomSheet.window?.setSoftInputMode(android.view.WindowManager.LayoutParams.SOFT_INPUT_ADJUST_RESIZE)
bottomSheetInternal?.let {
    val behavior = BottomSheetBehavior.from(it)
    it.layoutParams.height = ViewGroup.LayoutParams.MATCH_PARENT
    behavior.state = BottomSheetBehavior.STATE_EXPANDED
}
```

### Жизненный цикл занятия
* Внедрена система сохранения состояния через SharedPrefs
* Занятие больше не сбрасывается при переключении между вкладками или сворачивании приложения
* Установлен жесткий лимит жизни сессии (1.5 часа), после которого интерфейс автоматически возвращается в исходное состояние

```kotlin
if (id != -1 && subject != null && (System.currentTimeMillis() - startTime) < LESSON_DURATION_MS) {
    currentLessonId = id
    updateUiOnLessonStart(subject, null)
} else {
    resetUiAfterLesson()
}
```

### Улучшение UX
* Добавлена принудительная прокрутка (NestedScrollView) в меню создания занятия для удобства работы на небольших экранах
* Кнопка "Завершить" теперь требует подтверждения через AlertDialog, чтобы преподаватель не мог случайно закрыть занятие и активный QR

```kotlin
btnFinishLesson.setOnClickListener { 
    AlertDialog.Builder(requireContext())
        .setTitle("Завершить занятие?")
        .setMessage("Вы действительно хотите завершить текущее занятие?")
        .setPositiveButton("Да") { _, _ -> finishCurrentLesson() }
        .setNegativeButton("Отмена", null)
        .show()
}
```
