# Поэтапное описание
## Список issues системы для конечных пользователей (студентов и преподавателей)

### Аппаратные ограничения

- [ ] **Не у всех студентов есть телефоны** Звучит как будто такого нет, но оно есть и это нужно учитывать (наверное)...
- [ ] **У кого-то разрядился телефон** Банально, но системно. Если отметка только через смартфон - студент без заряда автоматически становится прогульщиком.
- [ ] **Не у всех Android** На IOS реализовать отметку через NFC или фоновые сервисы это невозможно(опираясь на research Вадима).

### Логические и организационные issues

- [ ] **QR-код не спасает от передачи** Достаточно один раз отсканировать и скинуть фото и человек из коридора (или своего дома) получит отметку.
- [ ] **Геолокация не панацея** Если фрод находится в том же здании (коридор, буфет, этажом выше), геолокация его не вычислит.
- [ ] **Уникальные QR это дорого** Генерация и валидация индивидуальных кодов на поток под 200+ человек создаёт реальную нагрузку на сервер и усложняет код. Вопрос что будет если такая поточка не одна.
- [ ] **Проблема логина** Ничто не мешает студенту зайти в аккаунт прогульщика со своего телефона и отметиться за него. Аккаунт может передаваться, если есть система логина.

### Проблемы с NFC

- [ ] **Можно случайно разбить телефон** В потоке из 200 человек кто-то обязательно неудачно приложится, выронит или стукнет телефон об телефон. Один инцидент перекроет всю экономию времени и бюджета.
- [ ] **Люди могут специально тупить** Искать чип, медленно подносить, переспрашивать это классика, чтобы "убить время" пары. Каждый студент по минуте и занятия не будет.
- [ ] **Физический износ и риски** Уронили, поцарапали камеру и тд - боль для преподавателя и студента одновременно, но при массовом использовании становится проблемой.

### Дополнительные риски

- [ ]  **Студенты с двумя телефонами** При наличии двух устройств один студент может физически находиться в аудитории и отмечать себя, а вторым телефоном (оставленным дома или переданным) друга. И ведь система не отличит...
- [ ] **Отсутствие интернета в аудитории** Если занятия в подвале или в плохих радиоусловиях, мобильная связь может не работать. Вся система отметок встанет на этом.


## CI/CD
На основе github actions был разработан workflow file который отслеживает ветку main и проверяет backend + frontend на соответствие swagger документации. Если всё сходится, то происходит деплой на сервер.
```yml
name: E-Journal CI/CD

on:
  push:
    branches: [ "main" ]
  pull_request:
    branches: [ "main" ]

jobs:
  # ==================================================
  # ДЖОБА 1: ПРОВЕРКА КОНТРАКТОВ (Frontend + Backend)
  # ==================================================
  validate:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - name: Install Frontend Dependencies
        run: npm ci

      - name: Contract Check (Frontend + Swagger)
        run: npm run check-api
      - name: Start Backend Services
        run: docker compose up -d ejournal-backend

      - name: Wait for API to be ready
        run: |
          echo "Ожидаем поднятия базы, выполнения миграций и старта API..."
          sleep 15 
      - name: Run Schemathesis (API Contract Testing)
        run: |
          docker run --network="host" \
            -v ${{ github.workspace }}/docs:/docs \
            schemathesis/schemathesis:3.39.4 run /docs/swagger.json \
            --base-url http://localhost:8888
  # ==========================================
  # ДЖОБА 2: ДЕПЛОЙ НА СЕРВЕР
  # ==========================================
  deploy:
    needs: validate
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to Server via SSH
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.SSH_HOST }}
          username: ${{ secrets.SSH_USER }}
          password: ${{ secrets.SSH_PASSWORD }}
          port: ${{ secrets.SSH_PORT }}
          script: |
            cd /home/sibsutis-ms/ejournal
            git pull origin main
            docker compose up -d --build
```
